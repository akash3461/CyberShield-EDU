# Contributing to CyberShield-EDU

First off, thank you for considering contributing to CyberShield-EDU! It's people like you that make CyberShield a great security tool for students.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## 2. Setting up your environment

Please refer to our [`docs/setup_guide.md`](docs/setup_guide.md) for full instructions on how to install dependencies and run both the FastAPI backend and the React frontend locally.

## 3. Pull Request Process

1.  Ensure any new Python code is PEP 8 compliant.
2.  Update the `README.md` and API docs with details of changes to the interface.
3.  Submit a Pull Request targeting the `main` branch. 
4.  You may merge the Pull Request in once you have the sign-off of at least one other developer.

## 4. Scam Datasets
If you are contributing to the AI/Heuristics:
Please ensure any scam URLs or texts added to the `data/` folder are properly anonymized if they contain PII (Personally Identifiable Information). Do not commit live, malicious executables to this repository.
